## Project Overview

This milestone establishes the foundation for a full-scale distributed data warehouse system.
We selected the **UK Road Safety: Traffic Accidents and Vehicles** dataset from Kaggle, covering
over **1.5 million accident records** from 2005–2014 across England, Scotland, and Wales. The
pipeline automates raw data downloading from Kaggle, performs pre-upload validation
(file size, encoding,row count), and ingests all validated files into **Hadoop HDFS** using a
structured, date-partitioned directory layout — ready for PySpark ETL in Assignment 3.

**HDFS Target Path:** `/warehouse/raw/uk_road_safety/year=YYYY/month=MM/`

## How to Run

### Step 1 — Run the Ingestion Pipeline
```bash
python ingest.py
```

**What it does automatically:**
1. **Load** — Downloads the UK Road Safety dataset from Kaggle via API
2. **Validate** — Checks each CSV: file size (MB), encoding (UTF-8), row count
3. **Organize** — Creates HDFS directory: `/warehouse/raw/uk_road_safety/year=YYYY/month=MM/`
4. **Upload** — Pushes all validated CSVs to HDFS using the `hdfs` Python library
5. **Log** — Records every step to `ingestion_pipeline.log` (successes, warnings, errors)

After running, check the log:
```bash
cat ingestion_pipeline.log
```

### Step 2 — Run the Profiling Notebook
```bash
jupyter notebook profiling_notebook.ipynb
```

Run all cells top-to-bottom. This will:
- Load all CSVs from `./raw_data/` (locally, not from HDFS — same data, faster pandas access)
- Perform all profiling analyses (schema, missing values, stats, distributions, quality checks)
- Save all chart figures to `./figures/` — these are included in the LaTeX report

### Step 3 — Compile the Profiling Report
```bash
# Run twice — first pass builds structure, second pass fixes references & ToC
pdflatex profiling_report.tex
pdflatex profiling_report.tex
```

This produces `profiling_report.pdf` — the submission document.
