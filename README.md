## Project Overview

Building on Assignment 2's ingestion and profiling work, this milestone implements:

1. **PySpark ETL Pipeline** (`etl.py`) — Transforms raw accident CSVs from HDFS, builds a
   star schema (1 fact table + 4 dimension tables), writes Parquet to HDFS, and validates output.
2. **Analytical Queries & Visualizations** (`analytics.ipynb`) — 5 Spark SQL business queries
   (including 2 with window functions and 1 time-based trend) plus 4 chart types.
3. **Pipeline Optimization** — Partitioning, broadcast joins, caching, and query plan analysis.

**Dataset:** UK Road Safety: Traffic Accidents and Vehicles (1.5M+ accident records, 2005–2014)
**HDFS Paths:**
- Raw: `/warehouse/raw/uk_road_safety/year=2026/month=04/`
- Processed: `/warehouse/processed/{fact_accidents,dim_time,dim_geography,dim_road,dim_environment}/`

## How to Run

### Step 1 — Run the ETL Pipeline
```bash
spark-submit etl.py
```

**What it does:**
1. Reads 3 accident CSVs from HDFS
2. Applies all 10 A2 cleaning transformations
3. Builds star schema (fact_accidents + 4 dimensions)
4. Writes Parquet to `/warehouse/processed/`
5. Validates row counts and null checks
6. Logs everything to `etl_pipeline.log`

### Step 2 — Run the Analytics Notebook
```bash
jupyter notebook analytics.ipynb
```

Run all cells top-to-bottom. This will:
- Load Parquet tables from HDFS
- Execute 5 Spark SQL business queries (BQ1–BQ5)
- Generate and save 4 charts to `figures/`
- Display query plan analysis for optimization

### Step 3 — Compile the Report
```bash
pdflatex final_report.tex
pdflatex final_report.tex   # Run twice for TOC
```

The report auto-fetches all charts from `figures/`.
